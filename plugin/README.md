# OwnedC Clang Plugin (Architecture Document)

Building a full C++ Clang compiler plugin requires the LLVM development headers and libraries to be installed locally (often absent on default macOS or Linux environments). 

For **Phase 4**, we implemented a robust Python-based analyzer in `tools/ownedc-analyzer.py` that mimics this functionality without the overhead. However, the files in this directory serve as a blueprint for a native LLVM plugin in the future.

## How the Native Plugin Works
The native Clang plugin relies on **AST Matchers**. AST Matchers allow us to query the Abstract Syntax Tree (AST) generated by Clang during compilation.

For example, our plugin matches nodes like:
- `callExpr(callee(functionDecl(hasName("malloc"))))`
- `varDecl(hasInitializer(callExpr(callee(functionDecl(hasName("owner_malloc"))))))`

When the compiler hits a matched node, it invokes our C++ callback where we can emit native compiler warnings and errors using Clang's `DiagnosticsEngine`. This deeply embeds OwnedC rules directly into `clang` or `clang-tidy`, meaning developers don't need a separate script or VSCode task—they get errors directly during compilation!

## Compiling the Plugin (Future)
When LLVM is installed (e.g., via `brew install llvm` or apt), you can compile the stub:

```bash
mkdir build && cd build
cmake -DLLVM_DIR=/path/to/llvm/lib/cmake/llvm -DClang_DIR=/path/to/llvm/lib/cmake/clang ..
make
```

To run it during a build:
```bash
clang -Xclang -load -Xclang ./libOwnedCClangPlugin.so -Xclang -plugin -Xclang ownedc-analyzer ...
```
